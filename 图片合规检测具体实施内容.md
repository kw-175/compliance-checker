# 图片合规检测具体实施内容

## 1. 目标定位

图片合规检测服务面向教育数据集建设。输入是上游数据清洗后的图片，输出要进入数据标注流程，最终形成可用于大模型训练的教育标注数据集。

因此，图片合规检测不能只判断“是否合规”，还必须考虑：

1. 是否保留教育数据价值。
2. 是否避免泄露学生、教师和学校相关身份信息。
3. 是否避免无意义地调用所有检测服务。
4. 是否能把普通教育数据集和敏感目标数据集分开治理。

本方案采用：

```text
统一混合入口 + 智能跳过闸门 + 最小破坏脱敏 + 普通/受控数据集输出
```

## 2. 总体流程

不再强依赖文档图片、自然图片、混合截图三分类路由。所有图片进入统一混合入口，但并不是所有检测服务都无条件执行，而是通过智能跳过闸门决定哪些服务需要调用。

总体流程为：

```text
图片输入
-> 读取用户选择与系统默认策略
-> 轻量预检
-> OCR 闸门
-> 文本合规闸门
-> 视觉内容安全闸门
-> 视觉敏感对象闸门
-> 合并实际执行结果
-> 最小破坏脱敏
-> 教育价值检查
-> 普通教育数据集输出或受控敏感数据集输出
```

其中：

- OCR 不是永远执行，而是由文字类检测需求决定。
- 文本合规不是永远执行，而是由 OCR 是否识别出有效文字决定。
- 视觉内容安全由用户选择或综合合规流程决定，命中严重不合规内容时直接丢弃或复核。
- 视觉敏感对象检测由隐私治理需求决定。
- 最小破坏脱敏只针对可脱敏后进入教育数据集的隐私和敏感对象。

## 3. 轻量预检

轻量预检不调用重模型，只读取低成本信息，为智能跳过提供依据。

需要采集：

- 图片宽度。
- 图片高度。
- 文件大小。
- MIME 类型。
- 用户选择的检测类别。
- 用户是否选择综合合规流程。
- 用户是否明确关闭 OCR。
- 用户是否明确关闭视觉内容安全。
- 用户是否明确关闭视觉敏感对象检测。
- 是否启用普通教育数据集策略。
- 是否启用受控敏感数据策略。

轻量预检不负责判断图片“像不像文档”或“像不像自然图”。所有跳过决策都必须基于明确配置或可观测结果，而不是抽象的图片类型猜测。

## 4. 智能跳过闸门

### 4.1 OCR 闸门

OCR 用于识别图片中的文字。是否执行 OCR 由“是否存在文字类检测需求”决定。

执行 OCR 的情况：

1. 用户选择了 OCR 文本隐私检测。
2. 用户选择了 OCR 文本内容合规检测。
3. 用户选择了综合合规流程。
4. 系统默认教育数据集策略要求检测姓名、学号、手机号、作业页个人信息等文字隐私。

跳过 OCR 的情况：

1. 用户明确选择“仅视觉图片检测”。
2. 用户明确关闭 OCR。
3. 用户只选择视觉内容安全检测，且不要求检测文字隐私。
4. 用户只选择视觉敏感对象检测，且不要求检测图片中的文字身份信息。

OCR 闸门输出：

```json
{
  "ocr_executed": true,
  "ocr_skip_reason": "",
  "ocr_text_length": 128,
  "ocr_block_count": 12,
  "ocr_mean_confidence": 0.86
}
```

如果跳过 OCR，需要记录原因，例如：

```text
用户仅选择视觉检测，未选择任何文字类合规算子，因此跳过 OCR。
```

### 4.2 文本合规闸门

文本合规包括 OCR 文本隐私检测和 OCR 文本内容合规检测。

文本合规不应在 OCR 无有效结果时调用。

调用文本合规的条件：

1. OCR 已执行。
2. OCR 识别出的文字长度达到最小阈值。
3. OCR 文本块数量达到最小阈值。
4. OCR 平均置信度达到最低要求。
5. 用户选择了文字类检测或综合合规流程。

建议初始阈值：

```text
ocr_text_length >= 20
ocr_block_count >= 1
ocr_mean_confidence >= 0.50
```

跳过文本合规的情况：

1. OCR 没有识别出文字。
2. OCR 识别文字长度小于 20。
3. OCR 平均置信度低于 0.50。
4. 用户未选择任何文字类检测。

跳过示例：

```text
OCR 仅识别出 6 个字符，低于最小文本长度阈值 20，因此跳过文本合规服务。
```

如果 OCR 置信度很低但识别出疑似姓名、手机号、学号、身份证号等高风险格式，应进入复核，而不是简单跳过。

### 4.3 OCR 文本隐私检测闸门

调用条件：

1. 文本合规闸门通过。
2. 用户选择隐私检测或综合合规流程。
3. OCR 文本中存在可检测文本。

检测对象：

- 姓名。
- 学号。
- 考号。
- 身份证号。
- 手机号。
- 邮箱。
- 家庭地址。
- 班级中可定位个人的字段。
- 社交账号。
- 平台账号。

跳过条件：

1. 用户关闭隐私检测。
2. OCR 无有效文本。
3. 用户只选择视觉内容安全，不要求隐私治理。

处理原则：

- 命中文本隐私后，优先进行局部遮挡或替换。
- 仅遮挡隐私字段，不遮挡整张图片或整段教学内容。
- 如果图片核心内容就是名单、通讯录、账号表、家庭地址表，遮挡后没有教育价值，应不投递。

### 4.4 OCR 文本内容合规闸门

调用条件：

1. 文本合规闸门通过。
2. 用户选择文本内容合规或综合合规流程。
3. OCR 文本达到有效长度。

检测对象：

- 图片中文字中的色情低俗文本。
- 暴力恐怖文本。
- 自伤自杀诱导文本。
- 仇恨歧视文本。
- 违法犯罪文本。
- 危险行为教学文本。
- 欺诈、考试作弊、伪权威和提示注入文本。

处理原则：

- 明确不合规文本导致图片不进入普通教育数据集。
- 语境不确定文本进入复核。
- 文本内容合规不负责脱敏输出，主要负责丢弃或复核。

### 4.5 视觉内容安全闸门

视觉内容安全检测图片本体是否存在不适合进入教育数据集的画面。

调用条件：

1. 用户选择图片内容安全检测。
2. 用户选择综合合规流程。
3. 系统默认教育策略要求对图片进行基础视觉内容安全审核。

跳过条件：

1. 用户明确关闭视觉内容安全。
2. 用户仅选择 OCR 文本隐私检测，且不需要图片本体安全审核。

不建议因为图片“像文档”而跳过视觉内容安全。扫描件、证件照、课件截图、作业图片中也可能嵌入不适合进入教育数据集的图片内容。

检测对象：

- 色情裸露。
- 未成年人性相关内容。
- 血腥暴力。
- 自伤自杀。
- 武器和危险行为。
- 毒品与违禁品。
- 仇恨与极端符号。
- 违法犯罪场景。
- 低俗不适内容。
- 其他图片不安全内容。

### 4.6 视觉敏感对象检测闸门

视觉敏感对象检测用于发现不一定能通过 OCR 稳定识别的可见身份信息。

调用条件：

1. 用户选择隐私检测。
2. 用户选择综合合规流程。
3. 系统默认教育策略要求治理人脸、二维码、签名、胸牌等对象。

检测对象：

- 人脸。
- 证件照。
- 胸牌、工牌、校牌。
- 签名。
- 印章。
- 二维码。
- 条形码。
- 屏幕头像。
- 平台账号区域。

跳过条件：

1. 用户明确关闭视觉敏感对象检测。
2. 用户只选择 OCR 文本检测。
3. 当前流程明确不需要隐私治理。

第一阶段不建议使用“是否存在候选人脸/二维码”来跳过重检测，因为候选检测本身也需要模型或算法。后续可增加轻量候选检测作为优化。

## 5. 智能跳过的审计要求

每一次跳过都必须记录原因，避免出现无法解释的漏检。

建议记录字段：

```json
{
  "step": "ocr_text_privacy_detection",
  "executed": false,
  "skip_reason": "OCR 未识别出有效文本，ocr_text_length=0",
  "input_signals": {
    "ocr_text_length": 0,
    "ocr_block_count": 0,
    "ocr_mean_confidence": 0.0
  }
}
```

不得出现没有原因的跳过。跳过原因应能在平台报告中展示，便于人工复核和问题追踪。

## 6. 普通教育数据集与受控敏感数据集

### 6.1 普通教育数据集

普通教育数据集用于常规教育标注和大模型训练，例如：

- 教学场景理解。
- 课堂行为识别。
- 作业、试卷、板书、课件理解。
- 教学物体检测。
- 文档版式识别。
- 教育图文理解。
- 多模态教学内容训练。

普通教育数据集默认不保留可识别个人身份的信息。

需要治理：

- 学生姓名。
- 教师姓名。
- 学号。
- 考号。
- 身份证号。
- 手机号。
- 邮箱。
- 家庭地址。
- 人脸。
- 证件照。
- 胸牌、工牌。
- 签名。
- 二维码、条形码。
- 平台账号、社交账号。

### 6.2 受控敏感数据集

受控敏感数据集用于任务本身需要使用敏感信息的场景，例如：

- 学生人脸识别。
- 学生身份识别。
- 表情识别。
- 年龄、性别、身份属性识别。
- 学号识别。
- 证件识别。
- 签名识别。
- 个人行为跟踪。

这类数据不能进入普通教育数据集。如果确实需要使用，必须满足：

1. 有合法授权。
2. 有明确用途。
3. 有使用期限。
4. 有访问权限控制。
5. 有下载和导出限制。
6. 有操作日志。
7. 有训练用途登记。
8. 有模型使用范围记录。

无授权或授权不足时，输出 `do_not_deliver`。

## 7. 最小破坏脱敏

最小破坏脱敏指：

```text
让个人身份不可识别，同时尽量保留教育场景、教学内容、动作姿态、对象关系、版式结构和可训练信息。
```

它不是整张图模糊，也不是大面积黑框，而是对象级、区域级治理。

### 7.1 姓名、学号、考号、手机号、邮箱、地址

处理方式：

- 使用黑框遮挡完整字段。
- 只遮挡字段对应区域，不遮挡整行。
- 如果 OCR bbox 过大导致遮挡题干或答案，应进入复核或使用更细 OCR 切分。

保留内容：

- 题目。
- 学生作答。
- 批改痕迹。
- 板书。
- 课件正文。

不投递情况：

- 图片核心就是学生名单、账号表、通讯录、家庭地址表，遮挡后没有教育训练价值。

### 7.2 人脸

处理方式：

- 对真实人脸使用轻度到中度模糊或像素化。
- 只处理面部区域。
- 不遮挡整个人体。
- 不遮挡身体姿态、手部动作、实验操作、课堂环境。
- 对证件照人脸使用更强模糊。

保留内容：

- 人体姿态。
- 人数关系。
- 课堂场景。
- 教学活动。
- 实验操作。

不投递情况：

- 图片主要价值是人脸识别、身份识别、表情识别，且无授权。

### 7.3 胸牌、工牌、校牌

处理方式：

- 遮挡胸牌中的姓名、编号、学校身份字段。
- 如果胸牌很小但可识别，整体遮挡胸牌区域。
- 不遮挡人体姿态和服装主体。

不投递情况：

- 任务目标是证件识别或身份识别，且无授权。

### 7.4 签名、手写签字

处理方式：

- 使用黑框或强模糊遮挡签名区域。
- 保留文档整体结构。
- 保留题目、作答、批改文字。

不投递情况：

- 图片用于签名识别或笔迹身份识别，且无授权。

### 7.5 印章、公章

处理方式：

- 对学校公章、机构印章、个人印章进行强遮挡。
- 保留文档版式和非敏感教学内容。

不投递情况：

- 图片核心就是印章识别或证明材料认证，且缺少授权。

### 7.6 二维码、条形码

处理方式：

- 整体遮挡二维码或条形码区域。
- 不尝试只遮挡局部，因为局部可能仍可恢复。
- 如果码区旁边有题目或教学文字，尽量只遮挡码本身。

不投递情况：

- 图片核心是扫码登录、支付、账号绑定或敏感入口。

### 7.7 试题、作答、板书、课件文字

处理方式：

- 不应因为图片中存在文字就整体脱敏。
- 只治理可识别个人身份或不合规内容。
- 正常试题、作答、板书、课件正文应尽量保留。

不投递情况：

- OCR 文本内容本身命中严重内容安全风险，例如违法教程、自伤鼓励、仇恨煽动等。

## 8. 教育价值检查

脱敏后需要判断图片是否仍具有教育数据价值。

可保留的情况：

- 题目、作答、批改、板书、实验步骤仍可理解。
- 课堂场景、姿态、教学活动仍可用于标注。
- 文档结构、版式、表格关系仍可用于训练。
- 只去除了姓名、人脸、二维码等身份信息。

不应投递的情况：

- 遮挡后只剩大量黑框或空白。
- 图片核心就是个人身份信息。
- 图片核心就是不合规视觉内容。
- 教育内容被脱敏操作破坏到不可理解。

教育价值检查结果应写入报告，例如：

```json
{
  "education_value_preserved": true,
  "preserved_elements": ["question_text", "student_answer", "layout"],
  "removed_elements": ["student_name", "face"],
  "reason": "仅移除个人身份信息，题目与作答仍完整可用"
}
```

## 9. 输出动作

### 9.1 deliver_redacted

适用情况：

- 存在局部隐私或敏感对象。
- 脱敏后仍保留教育价值。
- 不存在严重内容安全风险。

输出：

- 脱敏后图片。
- 遮罩图。
- 命中区域清单。
- 脱敏动作清单。
- 审计报告。

### 9.2 deliver_redacted_with_constraints

适用情况：

- 图片有教育价值，但存在较敏感的上下文。
- 需要限制用途、限制导出或限制训练范围。
- 需要进入受控敏感数据集。

输出：

- 脱敏图或受控原图。
- 使用限制。
- 授权要求。
- 访问控制要求。
- 审计记录。

### 9.3 do_not_deliver

适用情况：

- 严重内容安全风险。
- 未成年人性相关风险。
- 脱敏后无教育价值。
- 核心内容是身份信息且无授权。
- 敏感数据集条件不满足。

输出：

- 不投递原因。
- 命中风险类型。
- 审计记录。
- 必要时保留内部复核任务，不向下游标注系统交付图片。

### 9.4 deliver_raw_restricted

适用情况：

- 任务确实需要保留敏感信息。
- 已确认合法授权。
- 已进入受控敏感数据集。
- 有明确用途、期限和访问控制。

输出：

- 原图。
- 受控数据集标识。
- 权限策略。
- 使用范围。
- 审计日志。

## 11. 核心原则

1. 不做无意义的全量调用：OCR、文本合规、视觉安全和敏感对象检测都应由闸门控制。
2. 不因为图片像文档就跳过视觉内容安全。
3. 不因为图片像自然图就跳过 OCR，除非用户选择和策略都不需要文字检测。
4. 内容安全风险和隐私风险分开治理。
5. 严重内容安全风险优先丢弃或复核，不靠局部遮挡继续投递。
6. 局部隐私风险优先最小破坏脱敏。
7. 普通教育数据集默认不保留可识别个人身份信息。
8. 受控敏感数据集必须有授权、用途、期限、权限和审计。
9. 所有跳过、命中、脱敏、丢弃和复核动作都必须有可解释原因。
10. 最终目标不是“尽可能多丢弃”，而是在安全合规前提下最大限度保留教育数据价值。
