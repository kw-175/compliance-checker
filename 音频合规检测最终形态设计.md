# 音频合规检测最终形态设计

## 一、文档目的

本文档用于定义音频合规检测的最终目标形态。

该目标形态服务于教育等专有领域数据集建设流程，目标不是单纯做“音频转文字后的文本审核”，而是要对清洗后的音频数据进行完整合规治理，确保：

- 音频中不存在会影响大模型训练的不合规隐私内容；
- 音频中不存在会影响大模型训练的不适宜内容安全风险；
- 音频在进入下游自动化标注流程时，不会因为隐私风险、文本语义风险或非文本内容安全风险而污染标注结果；
- 最终保留下来的音频及其转写文本，能够稳定用于教育领域大模型训练与数据集生产。

因此，音频合规检测的最终目标，不是只判断“转写文本有没有问题”，而是要同时回答以下问题：

1. 音频转写文本中是否存在隐私风险或文本语义风险。
2. 音频声音本体中是否存在无法稳定转写、但本身不适合进入训练数据集的非文本内容安全风险。
3. 这些风险在当前上下文中，到底是在传播风险，还是在讨论、引用、警示、教学风险。
4. 整段音频整体上，是否适合进入教育领域训练数据集与自动标注流程。
5. 如果不适合直接进入，应当进入什么样的处理、复核或排除流程。

本文档不从“当前已实现到哪一步”出发，而是从“最终应当如何设计”出发，统一定义：

- 音频合规检测的总原则；
- 音频合规检测的核心流程；
- 转写文本合规与非文本内容安全的双通道设计；
- 片段级裁决、整段级判断、处理决策、人工复核与审计要求。

## 二、总体设计原则

音频合规检测不应等价于“音频先转文字，再复用文本合规”。

最终形态必须满足以下原则：

1. 音频合规必须同时覆盖两类风险：
   - 转写文本中的隐私与内容风险；
   - 声音本体中无法稳定转写的非文本内容安全风险。
2. 音频合规必须同时保留三类证据：
   - 原始音频可播放证据；
   - 转写文本可读证据；
   - 命中风险的时间段证据。
3. 不能只做片段级判断，还必须做整段音频级综合判断。
4. 不能只输出标签和编码，必须输出用户可读的自然语言说明，讲清楚为什么合规、为什么不合规、为什么需要处理或复核。
5. 自动化结果必须允许用户人工审核与修改，所有修改必须留痕可审计。
6. 对高风险、强操作性、未成年人有害、可执行危险内容以及明显不适宜教育场景的非文本声音内容，默认从严。
7. 对教学引用、历史分析、法律警示、安全教育、心理援助等保护性语境，允许通过上下文分析做减量，但前提是有充分证据说明该音频不是在传播风险。

## 三、音频合规检测与文本合规检测的关键差异

音频合规与文本合规有继承关系，但不能简单照搬。

### 3.1 音频比文本多出“非文本内容安全风险”

文本风险主要体现在文字语义中。

音频风险除了体现在“说了什么”，还体现在“声音本身呈现了什么非文本内容”。

例如：

- 呻吟、娇喘、暧昧喘息；
- 明显惊恐叫喊、痛苦呻吟、虐待线索；
- 枪声、爆炸声、打斗声、拍打声等危险环境声；
- 明显不适合教育场景的挑逗性、低俗化、煽动性声音表现；
这些内容即使无法稳定转写为文本，也可能已经不适合进入训练数据集。

### 3.2 音频合规必须以时间轴为基础

文本合规以字符偏移为核心。

音频合规必须同时管理：

- 文本字符偏移；
- 音频开始时间与结束时间；
- 说话人；
- 所属片段编号；
- 转写置信度。

否则就无法支撑：

- 音频播放器定位；
- 风险片段试听；
- 音频局部静音、蜂鸣、裁剪；
- 用户逐段审核与修改。

### 3.3 音频合规是双通道检测，不是单通道检测

音频合规最终形态必须同时包含：

1. 转写文本合规通道  
   基于 ASR 转写文本，执行隐私合规与内容安全合规。

2. 非文本内容安全通道  
   不依赖 ASR 文本，直接判断音频本体中无法稳定转写的内容安全风险。

最终再通过统一决策层做融合判断。

## 四、最终目标下的核心流程

音频合规检测的核心主线应当是：

`音频接入与标准化 -> 双通道风险分析 -> 片段级上下文裁决 -> 整段音频级综合判断 -> 处理决策 -> 用户可视化复核与修改 -> 合规音频输出`

其中“双通道风险分析”进一步拆为：

`转写文本合规通道 + 非文本内容安全通道`

下面对这套设计做详细定义。

## 五、双通道总体设计

### 5.1 转写文本合规通道

这条通道负责处理“可说成文字”的风险。

核心步骤为：

- 音频标准化
- ASR 转写
- 说话人分离
- 时间轴对齐
- 在转写文本上执行：
  - 音频隐私合规
  - 音频内容安全合规
- 再把文本命中片段映射回音频时间段

这条通道可复用你已经完成的文本隐私合规与文本内容安全合规能力。

### 5.2 非文本内容安全通道

这条通道负责处理“即使没有清晰转写文本，也本身不适合进入训练数据集”的非文本内容安全风险。

它不依赖 ASR 成功与否，而是直接分析：

- 声音类型
- 情绪强度
- 非语言发声
- 环境事件声
它应重点覆盖以下类型：

- 呻吟、娇喘、暧昧喘息等低俗化非语言发声
- 惊恐、求救、受害、虐待等高敏感场景音
- 枪声、爆炸声、打斗声、拍打声等危险环境声
- 明显不适合教育场景的挑逗性、刺激性、煽动性声音表达

#### 非文本通道的默认保守原则

非文本内容安全通道比转写文本通道更容易受到场景误判影响，因此必须采用保守治理原则。

系统不能因为某段声音“可能”属于体育活动、剧情表演、医学场景，就直接把它判定为可保留。

正确原则应当是：

1. 非文本风险一旦被命中，默认进入治理流程，不因缺少保护性上下文而直接放行。
2. 只有在附近转写文本、整段音频场景或人工确认能够提供充分保护性解释时，才允许将该片段从默认治理范围中减除。
3. 如果保护性证据不足，则应进入人工复核或更严格的限制性处理，而不是自动放行。

### 5.3 统一融合原则

两条通道都不是辅助通道，二者应当并行存在，并在统一决策层融合。

融合时必须遵循：

1. 转写文本风险与非文本内容安全风险任一成立，都不能因为另一通道“未命中”而直接放行。
2. 非文本高风险成立时，不能仅因转写文本安全就判定整体音频合规。
3. 转写文本风险命中但非文本片段具有保护性解释时，可以进入人工复核，而不是简单阻断。
4. 双通道同时命中时，应按更严格的处理决策执行。
5. 双通道结论冲突时，应在最终报告中明确说明“是哪一类证据主导了最终结论”，不能只给一个黑盒结果。

## 六、流程步骤与详细内容

### 6.1 音频接入与标准化

#### 目标

将清洗后的音频组织成可分析、可定位、可审计的输入对象。

#### 输入

- 清洗后的音频文件
- 来源信息
- 数据集构建任务信息
- 场景信息
- 已知元信息，如受众、来源、用途、项目要求

#### 处理内容

1. 音频文件摄取与来源登记。
2. 文件类型识别与 MIME 校验。
3. 统一格式、采样率、声道数和时长信息。
4. 生成 `audio_id`、文件哈希、来源路径、批次标识。
5. 为后续风险片段定位建立音频时间轴基线。

#### 输出

- 标准化音频对象
- 基础元信息
- 音频时间轴结构

### 6.2 音频级背景上下文构建

#### 目标

在进入双通道分析前，先判断整段音频整体是什么类型的材料。

#### 需要识别的核心信息

系统应尽可能识别：

- 音频类型：课堂录音、家校沟通、访谈、新闻、案例讲解、剧情表演、公开节目、混剪内容等
- 场景类型：教育场景、公开传播场景、内部记录场景、案例教学场景
- 受众对象：学生、未成年人、成年人、教师、混合受众
- 音频功能：讲解、转述、采访、互动问答、情绪表达、危险教程、攻击性表达、表演内容

#### 输出

- 音频级背景上下文对象
- 整段音频整体功能与语境摘要

### 6.3 说话人分离与时间轴对齐

#### 目标

为所有后续风险判断提供稳定的“谁在什么时候说了什么”以及“什么时候出现了什么声音事件”的定位能力。

#### 处理内容

1. 说话人分离。
2. ASR 段落与时间戳对齐。
3. 构建转写片段级 `TranscriptUnit`。
4. 建立音频时间段与文本片段之间的映射关系。

#### 输出

- 说话人片段
- 带时间戳的转写片段
- 音频与文本的双向定位索引

### 6.4 转写文本合规通道

#### 目标

在转写文本上执行与文本合规相同目标的风险识别与裁决。

#### 处理内容

1. 对转写文本执行隐私信息片段扫描。
2. 对转写文本执行文本内容安全风险片段扫描。
3. 结合说话人、时间轴、局部上下文和整段音频背景上下文做片段级分析。
4. 把命中结果映射回音频时间段。

#### 这条通道应覆盖的风险

至少包括：

- 姓名、联系方式、证件号、地址、学号、家长信息等隐私风险
- 政治敏感、色情低俗、暴力恐怖、仇恨歧视、辱骂霸凌、自伤自杀、违法教程、未成年人有害、误导欺诈、教育适宜性、提示注入与越狱等文本内容风险

#### 输出

- 带文本片段与音频时间段的转写文本风险列表

### 6.5 非文本内容安全通道

#### 目标

直接分析音频声音本体中的非文本内容安全风险，不依赖 ASR 文本是否成功转写。

#### 这一层为什么必须存在

如果没有音频本体通道，系统只能知道“说了什么”，却无法判断：

- 音频里是否存在明显低俗化非语言发声；
- 音频里是否存在惊恐、虐待、暴力环境声；
- 音频里是否存在仅靠声音就不适合教育训练的内容。

#### 非文本内容安全风险应重点覆盖的类型

建议至少包括：

- 呻吟、娇喘、暧昧喘息等低俗化非语言发声
- 惊恐叫喊、求救、痛苦呻吟、疑似虐待线索
- 枪声、爆炸、打斗、拍打等危险环境声
- 其他明显不适合教育场景的刺激性、挑逗性、攻击性声音表达

#### 非文本内容安全检测结果应输出什么

每个命中片段至少应包含：

- `event_id`
- `audio_id`
- `risk_type`
- `start_time`
- `end_time`
- `confidence`
- `initial_explanation`

如果能够进一步提取，也可包含：

- 说话人
- 伴随环境声类型
- 情绪强度
- 非语言声音模式

### 6.6 双通道片段级上下文裁决

#### 目标

无论风险来自转写文本还是非文本内容安全通道，最终都必须结合上下文进行片段级裁决。

#### 对转写文本片段的裁决

应参考文本隐私合规与文本内容安全文档中的原则，重点分析：

- 局部文本语义
- 邻近文本片段
- 说话人角色
- 音频整体场景
- 是否面向学生或未成年人
- 是否适合训练与自动标注

#### 对非文本风险片段的裁决

应重点分析：

1. 音频局部上下文
   - 命中时间段前后是否连续出现同类声学特征
   - 是否是孤立噪声、环境误判还是连续明确风险信号

2. 邻近转写文本上下文
   - 附近转写文本是否属于医学、体育、剧情表演、课堂示例、安全警示等保护性语境
   - 附近转写文本是否出现低俗、挑逗、暴力、胁迫等强化语义

3. 音频整体上下文
   - 整段音频是否为课堂录音、节目片段、剧情表演、网络传播内容、混剪内容等

4. 训练影响判断
   - 该声音内容是否会让模型学习到不适宜的音频风格或不适宜的多模态对应关系
   - 该非文本内容是否会污染教育场景的多模态训练样本

#### 片段级解释说明必须如何写

无论来自哪条通道，用户看到的说明都必须是自然语言，而不是标签堆叠。

不推荐写法：

- `risk_type = erotic_non_speech`
- `context_type = other`
- `decision = reject`

推荐写法：

- “该时间段虽然没有明确可转写文本，但存在连续呻吟与急促喘息声，且附近没有课堂讲解、医学说明或剧情表演语境作为保护性解释，因此该片段被判定为不适合进入教育训练数据集的非文本内容安全风险。”

或：

- “该片段命中了危险教程相关文本，但结合前后语境可见，音频整体是在进行安全教育讲解，目的是说明风险而不是鼓励实施。不过由于仍保留了较强的操作细节，因此建议进入人工复核，而不是直接放行。”

### 6.7 整段音频级综合判断

#### 目标

避免只看单个片段，忽视整段音频在整体结构和用途上的风险。

#### 需要综合判断的内容

1. 转写文本风险是否形成整段级组合风险。
2. 非文本内容安全风险是否在多个时间段重复出现并构成整体不适宜内容。
3. 转写文本风险与非文本内容安全风险是否相互强化。
4. 整段音频整体上是在传播风险，还是在讨论、教学、警示风险。
5. 整段音频是否适合作为教育领域训练样本。
6. 整段音频是否适合进入自动标注流程。

#### 输出

整段音频级结果至少应包含：

- 整体立场
- 整体风险等级
- 是否存在双通道组合风险
- 是否适合进入训练数据集
- 是否适合进入自动标注流程
- 是否需要人工复核
- 整段音频级口语化解释说明

## 七、处理决策生成

### 7.1 决策目标

将双通道检测与上下文裁决结果，转化为可执行的数据治理决策。

### 7.2 内容处理动作与流程处置动作必须分层

#### 第一层：内容处理动作

回答“音频内容本身怎么处理”：

- `保留`
- `转写脱敏`
- `转写泛化`
- `音频局部静音`
- `音频局部蜂鸣`
- `音频局部裁剪`
- `整段隔离`

#### 处理后音频与转写的一致性要求

音频合规中的内容处理动作不能只改音频，不改索引。

只要执行了局部静音、局部蜂鸣、局部裁剪或整段隔离，就必须同步维护以下一致性：

1. 处理后的音频时间轴是否变化。
2. 处理后的转写文本是否仍保留原始文本，还是需要输出“已处理版本”。
3. 风险片段的开始时间、结束时间、说话人、片段编号是否仍然有效。
4. 下游自动标注系统使用的是“原始时间轴”还是“处理后时间轴”。

建议默认规则如下：

- 局部静音、局部蜂鸣：
  不改变时间轴，但必须在结果中明确标记该时间段已处理。
- 局部裁剪：
  会改变时间轴，必须同步生成“处理后时间轴映射表”，否则不应直接进入自动标注链路。
- 转写脱敏、转写泛化：
  不改变音频，但必须保留原始转写与处理后转写的对应关系。

#### 第二层：流程处置动作

回答“这条音频怎么流转”：

- `正常流转`
- `受控流转`
- `人工复核`
- `排除训练`
- `仅评测或审计用途`

### 7.3 典型决策类型

#### 情况一：可直接进入数据集

适用场景：

- 双通道均未发现实质性风险；
- 或命中风险片段，但上下文明确属于教学、批判、警示、案例讲解、历史分析等保护性语境；
- 不会污染训练数据集与自动标注流程。

处理：

- 内容处理：`保留`
- 流程处置：`正常流转`

#### 情况二：处理后可进入数据集

适用场景：

- 存在可治理风险；
- 风险主要集中于局部时间段或局部转写片段；
- 经转写脱敏、局部静音、局部蜂鸣、局部裁剪后仍可保留音频主要教育价值。

处理：

- 内容处理：按风险类型执行局部治理
- 流程处置：`正常流转` 或 `受控流转`

#### 情况三：需人工复核

适用场景：

- 转写文本与非文本内容安全结果冲突；
- 保护性语境存在但不足以自动放行；
- 剧情表演、案例教学与风险传播边界不清；
- 非文本内容无法仅凭自动化结果稳定定性。

处理：

- 内容处理：输出机器建议
- 流程处置：`人工复核`

#### 情况四：不可进入训练数据集

适用场景：

- 存在高风险隐私内容；
- 存在明确不适宜教育场景的高危文本内容；
- 存在明显低俗化、挑逗性、虐待性、极端危险的非文本内容安全风险；
- 即使局部处理后，整段音频仍然不适合作为训练样本。

处理：

- 内容处理：可给出局部治理建议，但不能自动视为已放行
- 流程处置：`排除训练`

## 八、对下游自动标注流程的兼容性要求

音频合规治理不仅要保证训练安全，还要保证不会破坏自动标注流程。

系统必须单独判断：

- 处理后的音频是否仍可用于自动标注
- 处理后的转写文本是否仍保持结构稳定
- 局部静音、蜂鸣、裁剪是否会破坏标注定位
- 某些高风险音频是否应直接禁止进入标注前预处理链路

尤其要注意：

- 时间轴不能丢失
- 转写片段编号不能紊乱
- 音频局部处理不能让后续标注系统无法定位风险区域

## 九、用户结果展示

### 9.1 平台必须同时展示三类证据

用户界面至少应同时展示：

1. 原始音频播放器
2. 完整转写文本
3. 风险命中时间段与命中说明

### 9.2 每个风险片段建议展示的内容

对于每一个风险片段，建议展示：

- 命中来源：`转写文本风险` 或 `非文本内容安全风险`
- 风险类型
- 开始时间、结束时间
- 说话人
- 对应转写文本或“无明确可转写文本”
- 上下文摘录
- 口语化解释说明
- 建议处理动作
- 建议流程处置
- 是否影响训练
- 是否影响自动标注

### 9.3 整段音频级建议展示的内容

- 整体合规结论
- 是否可进入训练数据集
- 是否可进入自动标注流程
- 风险汇总
- 双通道组合风险说明
- 最终结论由哪一类证据主导
- 处理后预览

## 十、用户人工审核与修改

### 10.1 用户必须能够修改的内容

用户至少应能够对每一个风险片段的自动化结果进行修改，包括：

- 风险是否成立
- 风险来源通道
- 风险类型
- 时间段范围
- 说话人
- 对应转写片段
- 上下文判断
- 是否属于保护性语境
- 内容处理动作
- 流程处置动作
- 是否允许进入训练数据集
- 是否允许进入自动标注流程
- 解释说明文本

### 10.2 用户还应能够进行的操作

- 新增遗漏风险片段
- 驳回误报风险片段
- 合并重复命中
- 调整时间段范围
- 调整整段音频最终结论

### 10.3 审核结果必须留痕

每次人工修改都必须记录：

- 修改前结果
- 修改后结果
- 修改人
- 修改时间
- 修改原因

## 十一、最终输出产物

最终形态下，系统至少应输出：

1. 音频标准化结果
2. 说话人分离结果
3. ASR 转写结果
4. 转写文本风险结果
5. 非文本内容安全风险结果
6. 双通道片段级上下文裁决结果
7. 整段音频级综合判断结果
8. 内容处理结果
9. 流程处置结果
10. 用户人工审核结果
11. 完整审计日志

这些结果共同服务于：

- 平台前端展示
- 人工复核
- 下游自动标注
- 最终训练数据集导出
- 合规审计追踪

## 十二、最终形态下的判断核心

音频合规检测的最终核心，不是“音频里有没有敏感文本”，而是以下四个问题：

1. 音频转写文本中有哪些隐私风险与文本内容风险。
2. 音频声音本体中有哪些无法稳定转写、但本身不适合进入训练数据集的非文本内容安全风险。
3. 这些风险在当前上下文中，到底是在传播风险，还是在讨论、警示、教学风险。
4. 对这些风险做何种处理后，既能消除训练风险，又尽量不破坏下游自动标注和数据价值。

因此，最终形态下的音频合规检测，必须是：

- 大模型负责高召回的转写文本分析；
- 音频模型或规则负责非文本内容安全风险识别；
- 系统对双通道结果做片段级和整段级综合判断；
- 最终形成可解释、可处理、可人工修正的统一治理结果。

## 十三、建议的最终结论体系

为了让前端和用户更易理解，最终对外展示不建议只用抽象代码，而应直接展示自然语言结论。

建议最终整段音频级结论使用以下四类：

1. `可直接进入数据集`
2. `处理后可进入数据集`
3. `需人工复核`
4. `不可进入训练数据集`

必要时再配套补充：

- 是否允许自动标注
- 是否允许进入训练
- 是否已完成处理
- 是否存在非文本内容安全风险

## 十四、结论

音频合规检测的最终形态，不应只是“音频先转文字，再复用文本合规”的单通道系统，而应是一套完整的双通道音频治理系统。

它必须完成以下闭环：

`音频接入 -> 双通道检测 -> 上下文裁决 -> 整段判断 -> 处理决策 -> 展示说明 -> 人工修正 -> 审计留痕 -> 合规输出`

只有做到这一点，系统才能真正保证：

- 说出来的不合规内容不会进入教育领域训练数据集；
- 没说出来、但声音本体就不合规的内容也不会被漏放；
- 音频隐私风险、文本语义风险、非文本内容安全风险能被统一治理；
- 自动化标注流程不会被不适宜音频污染；
- 用户能够理解、复核并纠正系统输出；
- 最终保留下来的音频真正适合作为教育领域大模型训练数据。
