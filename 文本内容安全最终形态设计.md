# 文本内容安全最终形态设计

## 一、文档目的

本文档用于定义文本内容安全检测的最终目标形态。

该目标形态服务于教育等专有领域数据集建设流程，目标不是做一个泛化的互联网文本审核器，而是要对清洗后的文本数据进行内容安全治理，确保：

- 数据中不存在会影响大模型训练的不适宜内容；
- 数据在进入下游自动化标注流程时，不会因为内容风险导致流程失控或污染标注结果；
- 最终保留下来的文本能够稳定用于教育领域大模型训练与数据集生产。

因此，文本内容安全检测的最终目标，不是单纯判断“这段话危险不危险”，而是回答以下更贴近数据集建设的问题：

1. 这段文本中是否存在内容安全风险片段。
2. 这些风险片段在当前上下文中，到底是在传播风险，还是在讨论、引用、警示、教学风险。
3. 这篇文本整体上，是否适合进入教育领域训练数据集。
4. 如果不适合直接进入，应当进入什么样的处理或复核流程。

本文档前半部分重点定义以下内容：

- 文本内容安全最终形态的总原则；
- 核心流程框架；
- 片段级语义上下文裁决的设计要求与输出形式。

## 二、总原则

文本内容安全检测的最终形态，应遵循以下总原则。

### 2.1 内容安全的核心不是“扫词”，而是“判语义立场”

文本内容安全与隐私合规不同。

隐私合规的核心在于识别个人信息并治理；而内容安全的核心在于判断文本在当前语境下到底是在：

- 传播风险；
- 鼓励风险；
- 提供可执行方法；
- 还是在讨论风险、批判风险、警示风险、教学风险。

因此，内容安全检测不能只看是否命中了某类关键词、标签或敏感片段，而必须进一步判断该内容在上下文中的真实语义角色。

### 2.2 上下文分析是主判定步骤，不是辅助步骤

在内容安全场景中，很多文本脱离上下文是无法直接得出合规结论的。

例如：

- 一段暴力描述，可能是鼓励伤害，也可能是历史材料中的客观叙述；
- 一段自伤相关表述，可能是在诱导模仿，也可能是心理援助场景中的求助表达；
- 一段“如何绕过系统限制”的文字，可能是提示注入攻击，也可能是安全课程中的防护案例。

因此，最终形态下的内容安全检测流程必须把上下文裁决作为主步骤，而不是把它放在扫描之后作为可选修正。

### 2.3 片段级判断和文档级判断必须同时存在

文本内容安全不能只做单片段裁决。

原因在于：

- 有些片段单独看问题不大，但整篇文本整体立场是鼓励、煽动、危险教程；
- 有些片段看起来高风险，但整篇文本实际上是反诈、法治教育、历史分析、安全警示；
- 有些文本的内容安全风险不来自单个句子，而来自全文结构、叙事目标和操作导向。

因此，最终形态下必须同时存在：

- 片段级语义上下文裁决；
- 文档级整体立场判断。

### 2.4 最终判断标准必须服务于训练数据集建设

文本内容安全的最终判断标准，不应停留在“平台审核是否违规”这一层，而应落到训练数据治理场景中。

也就是说，系统最终要回答的不是：

- “它是不是一个风险话题”

而是：

- “它能不能进入教育领域训练数据集”
- “它能不能进入自动标注流程”
- “它是否只能进入人工复核或受限使用流程”

因此，最终输出必须与数据集流转、训练准入、自动标注适配性直接挂钩。

### 2.5 自动化结果必须可解释、可复核、可修改

文本内容安全检测不应只给出模型结论。

最终形态下，系统必须做到：

- 每个风险片段都能说明为什么这样判；
- 每篇文档都能说明整体上为什么可保留、需限制、需复核或不可进入训练；
- 用户能够对自动化结果进行人工审核与修改；
- 所有修改都可追踪、可审计。

### 2.6 高风险默认从严，保护性语境允许减量

对于以下内容，系统应默认从严：

- 高危可执行教程
- 未成年人有害内容
- 自伤鼓励或方法指导
- 明确仇恨煽动
- 实用性提示注入与越狱攻击
- 明显传播性、鼓励性、操纵性表达

对于以下语境，系统允许通过上下文裁决减量：

- 教学引用
- 历史分析
- 法律警示
- 安全教育
- 心理援助
- 反诈教育
- 批判性案例分析

但减量的前提必须是：上下文足以说明它不是在传播风险，而是在说明、分析、提醒或防范风险。

### 2.7 给用户的说明必须是自然语言，而不是标签堆叠

内容安全的上下文分析结果不能只以结构化标签呈现，例如：

- `context_type = teaching_reference`
- `risk_level = C2`
- `decision = P3`

这些字段适合系统存储，但不适合用户理解。

最终展示给用户的内容安全说明，必须写成自然语言，明确讲清：

- 风险片段出现在哪种上下文中；
- 该上下文说明它是在传播风险还是在讨论风险；
- 为什么因此被判定为可保留、需复核或不可进入训练；
- 这会对训练数据集和自动标注流程造成什么影响。

## 三、核心流程

文本内容安全检测的最终核心主线应当是：

`风险片段召回 -> 片段级语义上下文裁决 -> 文档级整体立场判断 -> 训练/标注适配决策 -> 用户可读解释生成 -> 人工复核覆盖 -> 最终合规输出`

这条主线的重点不是“召回了多少标签”，而是“是否真正理解了文本的风险语义与训练适配性”。

下面先定义前半部分的核心流程。

### 3.1 数据接入与文本标准化

#### 目标

将清洗后的文本数据组织成可稳定分析、可精确定位、可追踪审计的输入对象。

#### 输入

- 清洗后的文本内容
- 来源信息
- 数据集构建任务信息
- 场景信息
- 已知元信息，如受众、来源、用途、项目要求

#### 处理内容

1. 读取文本并构建标准化文档对象。
2. 生成 `doc_id`、文本哈希、来源标识、批次标识。
3. 建立字符偏移体系，保证后续所有风险片段都能精确定位。
4. 归一化元信息字段，为后续上下文分析提供基础输入。

#### 输出

- 标准化文档对象
- 文本元信息
- 可定位的字符偏移结构

### 3.2 文档级背景上下文构建

#### 目标

先识别文档整体是什么类型的文本，为后续片段级裁决提供背景语境。

#### 需要识别的核心信息

系统应尽可能识别：

- 文档类型：教材、学生作文、聊天记录、历史材料、新闻、法治教育、反诈教育、安全课程、案例分析等
- 场景类型：教育场景、公共传播场景、内部讨论场景、公开案例场景
- 受众对象：学生、未成年人、成年人、教师、混合受众
- 文本功能：讲解、叙述、转述、引用、警示、鼓励、教学、攻击、教程

#### 作用

文档级背景上下文的作用不是直接给出结论，而是为后续裁决提供一个判断框架。

例如：

- 如果整篇文本是安全教育材料，则某些高风险词不能直接等同于不合规传播；
- 如果整篇文本是面向未成年人的互动文本，则某些中风险表达应从严判断；
- 如果整篇文本明显是可执行危险教程，则片段级解释空间应明显收窄。

#### 输出

- 文档级背景上下文对象
- 文档整体功能与语境摘要

### 3.3 风险片段召回

#### 目标

由大模型优先完成高召回风险片段识别，尽可能找出所有疑似内容安全风险片段。

#### 为什么要先做高召回

内容安全片段的表达形式高度灵活：

- 有的通过直接词汇表达；
- 有的通过隐喻、改写、分步骤描述表达；
- 有的只在局部句子里显露风险；
- 有的虽然话题敏感，但语义立场并不危险。

因此，第一步应优先确保“不漏召回明显风险候选”，而不是一开始就追求高精确率。

#### 风险片段召回应输出的内容

每一个疑似风险片段至少应包含：

- `fragment_id`
- `doc_id`
- `label`
- `risk_type`
- `text`
- `start`
- `end`
- `severity`
- `confidence`
- `initial_explanation`

#### 召回结果的性质

这一层输出的是“疑似风险候选片段”，不是最终合规结论。

也就是说：

- 被召回，不等于最终不合规；
- 没有被召回，才意味着当前系统未发现明显内容安全风险证据。

### 3.4 片段级语义上下文裁决

#### 目标

对每一个被召回的风险片段，结合其局部语义和背景上下文，判断它到底是在传播风险，还是在讨论、引用、说明、警示、教学风险。

这是文本内容安全最终形态中最关键的步骤。

## 四、片段级语义上下文裁决

### 4.1 为什么片段级裁决是核心

风险片段召回只能告诉系统：

- “这里可能有风险内容”

但它不能直接回答：

- “这里是否真的构成不适合进入训练数据集的内容风险”

例如一句“炸药的制作步骤如下”：

- 如果它出现在实际教程中，属于高风险可执行内容；
- 如果它出现在安全教育材料中，仍然可能不适合直接进入训练集，但理由与处理方式不同；
- 如果它出现在新闻转述中，则需要判断是否保留了过强的操作细节。

因此，片段级裁决的任务不是简单确认标签，而是判断：

- 该片段的语义立场；
- 该片段的操作性强弱；
- 该片段的传播风险强弱；
- 该片段是否适合训练；
- 该片段是否需要人工复核。

### 4.2 片段级裁决要分析什么

每一个风险片段都应从以下几个维度做语义裁决。

#### 1. 局部语言功能

判断该片段在当前句子中到底是在做什么：

- 直接陈述
- 引用他人说法
- 转述案例
- 提供方法
- 提供步骤
- 表达态度
- 提出问题
- 进行批判
- 进行警示
- 进行教学说明

这一层决定它是“内容本体”，还是“对内容的讨论”。

#### 2. 风险立场判断

判断文本对该风险内容持什么立场：

- 鼓励
- 诱导
- 中性介绍
- 反对
- 警示
- 批判
- 防范
- 求助

这一步是内容安全和纯关键词过滤最大的区别。

同样一句涉及暴力、自伤、仇恨、违法教程的话，在不同立场下，训练风险完全不同。

#### 3. 操作性判断

判断该片段是否具有可执行性。

重点看是否包含：

- 明确步骤
- 工具名称
- 参数细节
- 实施顺序
- 规避方法
- 提示词模板
- 可直接照搬的表达

操作性越强，内容越不适合直接进入训练数据集。

#### 4. 受众影响判断

判断该片段如果被模型学习，是否会对教育场景造成额外风险。

尤其要判断：

- 是否面向未成年人
- 是否会诱导学生模仿
- 是否会强化错误价值导向
- 是否会污染教育场景对话风格

#### 5. 语境保护性判断

判断该片段是否处于可减量的保护性语境中，例如：

- 安全教育
- 法律警示
- 历史分析
- 反诈教学
- 心理援助
- 风险案例讲解

但需要注意：
保护性语境不自动等于可放行，只意味着它可能从“直接排除”转为“需复核”或“限制性保留”。

### 4.3 片段级裁决应输出什么

每一个风险片段在完成语义上下文裁决后，不应只保留标签，而应输出完整裁决结构。

至少应包含：

- 片段是否构成真实内容安全风险
- 风险是在传播、鼓励、指导，还是在讨论、引用、警示、教学
- 风险的操作性强弱
- 是否存在未成年人场景加严因素
- 是否属于保护性语境
- 是否适合直接进入训练数据集
- 是否适合进入自动标注流程
- 是否需要人工复核
- 口语化解释说明

### 4.4 解释说明必须如何写

片段级上下文裁决的结果，不能只以标签和布尔值展示给用户。

不推荐写法：

- `context_type = safety_education`
- `stance = warning`
- `operational = medium`
- `decision = P3`

推荐写法应当是：

- “该片段涉及违法危险内容，但它出现在安全教育语境中，文本目的是讲解风险而不是鼓励实施。不过该片段仍保留了较强的操作性描述，因此不适合直接进入训练数据集，建议进入人工复核。”

也就是说，解释说明必须回答：

1. 该片段出现在哪种上下文中。
2. 这个上下文说明它是在传播风险还是在讨论风险。
3. 为什么因此被判定为可保留、需复核或不可进入训练。
4. 这个判断对训练和自动标注意味着什么。

### 4.5 片段级裁决的结果类型

片段级裁决建议先统一归为以下四类：

#### 类型一：可直接保留

适用场景：

- 片段虽然命中风险标签，但上下文明确表明其为教学、批判、警示、历史分析或安全教育内容；
- 且其表达不具备明显传播性、鼓励性或强操作性。

#### 类型二：限制性保留

适用场景：

- 片段属于保护性语境，但仍保留一定风险表达；
- 可以保留用于受限训练、受限评测或结构化标注准备；
- 不适合直接作为普通训练样本放行。

#### 类型三：需人工复核

适用场景：

- 语境复杂；
- 引用与传播难区分；
- 教学与危险教程边界不清；
- 是否适合教育训练集存在明显不确定性。

#### 类型四：不可进入训练数据集

适用场景：

- 明确传播、鼓励、指导高风险内容；
- 具备较强可执行性；
- 涉及未成年人有害、高危暴力、自伤鼓励、仇恨煽动、实用越狱等高风险表达；
- 即使在部分上下文中出现，也不适合被模型学习。

### 4.6 片段级裁决的底线原则

片段级语义上下文裁决必须坚持以下底线：

1. 不能因为命中敏感词就直接判定不合规。
2. 也不能因为出现了“教学”“历史”“案例”等词就自动放行。
3. 必须同时看语义立场、操作性、受众影响和保护性语境。
4. 对高风险可执行内容和未成年人有害内容，应优先从严。
5. 片段级裁决的输出必须服务于后续文档级整体立场判断，而不是直接替代文档级结论。

## 五、文档级整体立场判断

### 5.1 为什么必须做文档级判断

即使片段级裁决已经完成，系统仍然不能直接得出最终内容安全结论。

原因在于，文本内容安全的很多风险并不是由单一片段决定的，而是由全文的整体叙述目标、立场走向、内容组织方式共同决定的。

例如：

- 某个片段只是中性叙述，但整篇文章是在系统性讲解如何规避监管；
- 某些片段单独看像教学引用，但整篇文本整体上是在合理化仇恨、暴力或危险行为；
- 某篇文章包含多个“看似可解释”的风险片段，但合起来形成了明确的诱导、煽动或模仿导向。

因此，片段级裁决只能回答“局部在说什么”，文档级判断必须回答“整篇文本到底在做什么”。

### 5.2 文档级整体立场判断的目标

文档级判断要识别的不是单个风险标签，而是全文的总体语义方向。

系统应判断：

- 全文是在传播、鼓励、指导高风险内容，还是在讨论、分析、警示、教学高风险内容；
- 全文整体是否具有明显的操作性和可模仿性；
- 全文是否适合进入教育领域训练数据集；
- 全文是否适合进入自动标注流程；
- 全文是否需要人工复核；
- 全文是否应直接排除出训练数据集。

### 5.3 文档级判断应重点分析的维度

#### 1. 文本整体目的

系统应判断整篇文本的主要目的是什么，例如：

- 教学说明
- 案例讲解
- 风险警示
- 历史叙述
- 新闻转述
- 操作教程
- 鼓动煽动
- 情绪发泄
- 价值引导
- 越狱提示传播

这一步决定全文的主立场，而不仅仅是局部句子的表面含义。

#### 2. 片段之间的组合关系

系统应判断多个风险片段之间是否形成结构化风险，例如：

- 多个分步骤片段组合成完整危险教程；
- 多个仇恨表达形成持续性攻击立场；
- 多个自伤相关表达组合成诱导式内容；
- 多个提示注入片段组成完整越狱模板。

也就是说，文档级判断要能识别“单片段不极端，但组合后高风险”的情况。

#### 3. 叙事导向与价值导向

系统应判断全文是否具有以下倾向：

- 合理化危险行为
- 鼓励违法违规
- 鼓励暴力或报复
- 美化自伤或极端行为
- 操纵、胁迫、羞辱、霸凌
- 强化不适合教育场景的价值导向

这类风险很多并不体现为“单句违规”，但会直接影响训练后模型的表达倾向。

#### 4. 受众场景加严

如果文档面向学生、未成年人或教育对话场景，则文档级判断必须从严。

例如：

- 某些在成人公共讨论中可进入复核的内容，面向学生场景时应直接限制或排除；
- 某些带模仿风险的表达，在未成年人场景下应显著提高风险等级；
- 某些粗俗、辱骂、极端功利导向内容，即使不属于传统高危标签，也可能不适合进入教育训练集。

#### 5. 教育领域训练适配性

文档级判断最终必须落到“是否适合作为教育领域训练样本”这一标准上。

也就是说，哪怕某篇文本不属于最严厉意义上的公开违规内容，只要它：

- 会污染教育场景表达风格；
- 会让模型学习到不适宜的危险指令结构；
- 会放大未成年人有害内容；
- 会给自动标注带来方向性误导；

那么它仍然不应直接进入目标训练数据集。

### 5.4 文档级判断应输出什么

文档级整体立场判断至少应输出：

- 文档整体立场
- 文档整体风险等级
- 是否存在组合风险
- 是否存在教育场景加严因素
- 是否适合进入训练数据集
- 是否适合进入自动标注流程
- 是否需要人工复核
- 文档级口语化解释说明

### 5.5 文档级解释说明应如何写

文档级解释说明不能只是“命中多个标签，因此高风险”。

推荐说明方式应类似：

- “该文本虽然包含安全教育语境，但全文不仅描述了风险现象，还保留了较完整的可执行步骤，因此整体上更接近危险教程而不是单纯警示材料，不适合直接进入训练数据集。”

或：

- “该文本命中了仇恨和攻击性表达，但结合全文内容可见，其目的是进行反歧视案例分析，并未鼓励传播相关立场。不过由于部分原文引用较强，仍建议进入人工复核后再决定是否保留。”

也就是说，文档级解释必须讲清：

1. 整篇文本整体上在做什么；
2. 为什么片段级风险在全文层面被增强、维持或减弱；
3. 为什么最终可保留、需复核或不可进入训练；
4. 对训练与自动标注的实际影响是什么。

## 六、训练/标注适配决策

### 6.1 这一层的目标

内容安全判断最终必须转化为数据治理决策。

系统不仅要知道“这篇文本有没有风险”，还要知道：

- 它能不能进入训练数据集；
- 它能不能进入自动标注流程；
- 它是否只能进入人工复核池；
- 它是否应该被直接排除。

因此，这一层的目标是把内容安全判断转化为可执行的数据流转决策。

### 6.2 内容结论与流程处置必须分层

和隐私治理一样，内容安全也不应把所有结论混成一个字段。

建议拆成两层：

#### 第一层：内容结论

回答“这篇文本从内容安全角度属于什么状态”：

- 可直接保留
- 限制性保留
- 需人工复核
- 不可进入训练数据集

#### 第二层：流程处置

回答“这篇文本接下来怎么流转”：

- 正常流转
- 受控流转
- 人工复核
- 排除训练
- 仅评测或审计用途

这样可以避免把“风险判断”和“流程动作”混在一起。

### 6.3 决策时必须考虑的因素

训练/标注适配决策至少应综合以下输入：

1. 片段级语义上下文裁决结果
2. 文档级整体立场判断结果
3. 文档是否面向学生或未成年人
4. 风险片段是否具有高操作性
5. 是否存在组合风险或全文系统性风险
6. 是否属于保护性教学语境
7. 是否会污染教育领域模型表达风格
8. 是否会破坏下游自动标注结构或标签判断

### 6.4 典型决策类型

#### 情况一：可直接进入数据集

适用场景：

- 命中风险片段，但上下文明确属于教学、批判、警示、法律分析、历史叙述等保护性语境；
- 风险表达不具备强传播性或强操作性；
- 文档整体适合教育训练语境；
- 不会对自动标注流程造成明显破坏。

处理：

- 内容结论：`可直接保留`
- 流程处置：`正常流转`

#### 情况二：限制性保留后进入数据集

适用场景：

- 文本具有一定风险表达，但仍有较强教育语义价值；
- 直接进入普通训练集不合适；
- 可以在受控条件下保留，例如仅进入受限池、受限训练流程或经过额外筛选后继续使用。

处理：

- 内容结论：`限制性保留`
- 流程处置：`受控流转`

#### 情况三：需人工复核

适用场景：

- 语境复杂，传播与引用边界不清；
- 保护性语境存在，但片段仍保留较强风险表达；
- 片段级和文档级判断之间存在冲突；
- 是否适合训练或自动标注无法稳定自动判断。

处理：

- 内容结论：`需人工复核`
- 流程处置：`人工复核`

#### 情况四：不可进入训练数据集

适用场景：

- 全文整体上属于高风险传播、鼓励、指导内容；
- 具备高操作性；
- 涉及未成年人有害、高危暴力、自伤鼓励、仇恨煽动、提示注入越狱等高危类别；
- 即使带有部分引用、讨论或案例语境，也不适合被模型学习。

处理：

- 内容结论：`不可进入训练数据集`
- 流程处置：`排除训练`

### 6.5 对自动标注流程的单独判断

某篇文本“不适合训练”，不等于它一定“不适合标注前处理”；
同样，某篇文本“可进入复核”，也不等于它可以直接进入自动标注流程。

因此，系统应单独给出：

- 是否允许进入自动标注流程
- 不允许时的原因

主要判断点包括：

1. 风险表达是否会干扰自动标签生成。
2. 风险内容是否会导致模型式预标注偏离教育场景目标。
3. 片段是否需要人工确认后才能安全进入标注。
4. 文本整体是否会让标注员或标注代理学习到不适宜的输出模式。

## 七、用户可读解释生成

### 7.1 为什么这一层必须单独存在

内容安全的判断过程天然复杂。

如果系统只输出：

- 命中标签
- 风险等级
- 处理动作

那么用户无法判断自动化结果是否可信，也无法有效进行人工复核。

因此，最终形态下必须把结构化判断转换成用户能直接理解的口语化说明。

### 7.2 解释说明应覆盖哪些内容

对于每个风险片段，说明应至少包含：

1. 片段出现在什么上下文中。
2. 这个上下文说明它是在传播风险还是在讨论风险。
3. 片段为什么被判定为可保留、限制性保留、需复核或不可进入训练。
4. 这个判断对训练数据集和自动标注流程意味着什么。

对于整篇文档，说明应至少包含：

1. 全文整体上在做什么。
2. 为什么全文整体风险被增强、维持或减弱。
3. 为什么最终允许流转、受控流转、人工复核或排除训练。

### 7.3 推荐说明风格

推荐说明应当是自然语言句子，而不是标签堆叠。

例如：

- “该片段涉及自伤相关表达，但上下文显示文本目的是提供心理援助与求助指引，而不是鼓励实施自伤行为，因此可作为保护性语境处理。不过考虑到该主题本身对教育训练有较高敏感性，建议进入人工复核。”

- “该文本命中了提示注入和绕过限制相关表达，且全文以操作步骤形式给出可直接使用的越狱方法，因此整体属于高风险实用攻击内容，不适合进入训练数据集，也不应进入自动标注流程。”

## 八、人工复核覆盖

### 8.1 人工复核不是补丁，而是正式环节

文本内容安全的很多场景天然存在边界模糊。

因此，人工复核不能被设计成异常补救，而必须被视为目标态中的正式步骤。

### 8.2 用户必须能够修改的内容

用户至少应能够修改：

- 风险标签
- 风险片段范围
- 片段级上下文判断
- 文档级整体立场判断
- 是否属于保护性语境
- 风险等级
- 是否适合训练
- 是否适合自动标注
- 最终流程处置
- 解释说明文本

### 8.3 审核结果如何覆盖自动结果

人工审核结果应当能够覆盖自动化判断，并形成新的最终决策。

覆盖后的结果必须明确标记：

- 原始机器决策
- 人工修改后的决策
- 修改原因
- 修改人
- 修改时间

这样才能保证平台上的最终结果既可解释，又可追踪。

## 九、最终输出与审计

### 9.1 系统最终应输出的结果

文本内容安全最终形态下，系统至少应输出：

1. 风险片段召回结果
2. 片段级语义上下文裁决结果
3. 文档级整体立场判断结果
4. 训练适配决策结果
5. 自动标注适配决策结果
6. 用户可读解释说明
7. 人工复核任务
8. 人工复核结果
9. 最终覆盖后的决策结果
10. 完整审计日志

### 9.2 审计日志应记录什么

审计日志至少应记录：

- 每个片段的初始召回依据
- 上下文分析结果
- 片段级裁决理由
- 文档级整体判断理由
- 训练与标注适配决策依据
- 人工修改前后差异
- 最终流转结果

## 十、建议的最终结论体系

为了让平台用户更容易理解，最终面向用户展示时，不建议只显示编码，而应直接显示自然语言结论。

建议最终文档级结论使用以下四类：

1. `可直接进入数据集`
2. `限制性保留后进入数据集`
3. `需人工复核`
4. `不可进入训练数据集`

同时可配套补充：

- 是否允许自动标注
- 是否允许进入训练
- 是否已完成人工复核
- 是否属于高风险类别

## 十一、结论

文本内容安全检测的最终形态，不应只是一个标签分类器，也不应只是一个敏感词拦截器。

它应当是一套完整的内容安全治理系统，完成以下闭环：

`召回 -> 理解 -> 片段裁决 -> 全文判断 -> 训练适配 -> 解释说明 -> 人工复核 -> 最终流转 -> 审计留痕`

只有做到这一点，系统才能真正保证：

- 高风险内容不会进入教育领域训练数据集；
- 保护性、教学性、分析性内容不会被粗暴误杀；
- 自动化标注流程不会被不适宜内容污染；
- 用户能够理解、复核并纠正系统输出；
- 最终保留下来的文本真正适合作为教育领域大模型训练数据。
